## Prerequisites

- A Yuno account with sandbox access
- Your API keys from the [Yuno Dashboard](https://dashboard.y.uno)

## Create Your First Payment

<Steps>
  <Step title="Get your API keys">
    Log in to the [Yuno Dashboard](https://dashboard.y.uno) and navigate to **Settings > API Keys**. You need three values:

    - `public-api-key` — Used for client-side SDK initialization and server-side calls
    - `private-secret-key` — Used for server-side API calls only
    - `account-code` — Your account identifier

    <Warning>
      Never expose your `private-secret-key` in client-side code or version control.
    </Warning>
  </Step>

  <Step title="Create a checkout session">
    Create a checkout session on your server to initialize the payment flow.

    <CodeGroup>
    ```bash cURL
    curl -X POST https://api-sandbox.y.uno/v1/checkout/sessions \
      -H "public-api-key: your-public-api-key" \
      -H "private-secret-key: your-private-secret-key" \
      -H "account-code: your-account-code" \
      -H "Content-Type: application/json" \
      -d '{
        "amount": {
          "currency": "USD",
          "value": 50.00
        },
        "country": "US",
        "merchant_order_id": "order-001",
        "payment_description": "Test payment",
        "account_id": "your-account-code"
      }'
    ```

    ```javascript Node.js
    const response = await fetch('https://api-sandbox.y.uno/v1/checkout/sessions', {
      method: 'POST',
      headers: {
        'public-api-key': 'your-public-api-key',
        'private-secret-key': 'your-private-secret-key',
        'account-code': 'your-account-code',
        'Content-Type': 'application/json',
      },
      body: JSON.stringify({
        amount: { currency: 'USD', value: 50.00 },
        country: 'US',
        merchant_order_id: 'order-001',
        payment_description: 'Test payment',
        account_id: 'your-account-code',
      }),
    });

    const session = await response.json();
    // Use session.checkout_session for SDK initialization
    ```

    ```python Python
    import requests

    response = requests.post(
        'https://api-sandbox.y.uno/v1/checkout/sessions',
        headers={
            'public-api-key': 'your-public-api-key',
            'private-secret-key': 'your-private-secret-key',
            'account-code': 'your-account-code',
            'Content-Type': 'application/json',
        },
        json={
            'amount': {'currency': 'USD', 'value': 50.00},
            'country': 'US',
            'merchant_order_id': 'order-001',
            'payment_description': 'Test payment',
            'account_id': 'your-account-code',
        },
    )

    session = response.json()
    # Use session['checkout_session'] for SDK initialization
    ```
    </CodeGroup>

    <Note>
      This endpoint returns HTTP **200** (not 201). The checkout session ID is returned in the response body.
    </Note>
  </Step>

  <Step title="Create a payment">
    Use the checkout session to create a payment.

    <CodeGroup>
    ```bash cURL
    curl -X POST https://api-sandbox.y.uno/v1/payments \
      -H "public-api-key: your-public-api-key" \
      -H "private-secret-key: your-private-secret-key" \
      -H "account-code: your-account-code" \
      -H "Content-Type: application/json" \
      -d '{
        "checkout_session": "your-checkout-session-id",
        "payment_method": {
          "type": "CARD"
        },
        "amount": {
          "currency": "USD",
          "value": 50.00
        },
        "country": "US",
        "customer": {
          "email": "customer@example.com"
        }
      }'
    ```

    ```javascript Node.js
    const payment = await fetch('https://api-sandbox.y.uno/v1/payments', {
      method: 'POST',
      headers: {
        'public-api-key': 'your-public-api-key',
        'private-secret-key': 'your-private-secret-key',
        'account-code': 'your-account-code',
        'Content-Type': 'application/json',
      },
      body: JSON.stringify({
        checkout_session: 'your-checkout-session-id',
        payment_method: { type: 'CARD' },
        amount: { currency: 'USD', value: 50.00 },
        country: 'US',
        customer: {
          email: 'customer@example.com',
        },
      }),
    });

    const result = await payment.json();
    ```

    ```python Python
    import requests

    response = requests.post(
        'https://api-sandbox.y.uno/v1/payments',
        headers={
            'public-api-key': 'your-public-api-key',
            'private-secret-key': 'your-private-secret-key',
            'account-code': 'your-account-code',
            'Content-Type': 'application/json',
        },
        json={
            'checkout_session': 'your-checkout-session-id',
            'payment_method': {'type': 'CARD'},
            'amount': {'currency': 'USD', 'value': 50.00},
            'country': 'US',
            'customer': {
                'email': 'customer@example.com',
            },
        },
    )

    payment = response.json()
    ```
    </CodeGroup>

    ### Regional Examples

    The example above uses a US card payment — the simplest case. Yuno supports local payment methods across the globe through a single API. Select a region to see how the payload adapts.

    <Tabs>
      <Tab title="Brazil — PIX">
        PIX is Brazil's instant payment system — real-time, 24/7, and the most popular payment method in the country. Requires a CPF (individual tax ID) or CNPJ (business tax ID).

        ```json
        {
          "checkout_session": "your-checkout-session-id",
          "payment_method": { "type": "PIX" },
          "amount": { "currency": "BRL", "value": 100.00 },
          "country": "BR",
          "customer": {
            "document": {
              "document_type": "CPF",
              "document_number": "12345678901"
            },
            "email": "cliente@example.com"
          }
        }
        ```

        Returns a QR code or copy-paste code. The customer pays via their banking app and you receive confirmation in seconds.
      </Tab>
      <Tab title="India — UPI">
        UPI (Unified Payments Interface) is India's dominant real-time payment method, processing billions of transactions monthly through apps like Google Pay, PhonePe, and Paytm.

        ```json
        {
          "checkout_session": "your-checkout-session-id",
          "payment_method": { "type": "UPI" },
          "amount": { "currency": "INR", "value": 2500.00 },
          "country": "IN",
          "customer": {
            "email": "customer@example.com",
            "phone": "+919876543210"
          }
        }
        ```

        UPI payments generate a payment link or QR code. The customer authorizes via their UPI app and you receive real-time confirmation via webhooks.
      </Tab>
      <Tab title="Europe — iDEAL & SEPA">
        **iDEAL** is the leading bank redirect method in the Netherlands (70%+ of online payments). **SEPA Direct Debit** covers all 36 SEPA countries for recurring EUR payments.

        ```json iDEAL (Netherlands)
        {
          "checkout_session": "your-checkout-session-id",
          "payment_method": { "type": "IDEAL" },
          "amount": { "currency": "EUR", "value": 75.00 },
          "country": "NL",
          "customer": {
            "email": "customer@example.com"
          }
        }
        ```

        ```json SEPA Direct Debit (EU-wide)
        {
          "checkout_session": "your-checkout-session-id",
          "payment_method": { "type": "SEPA_DIRECT_DEBIT" },
          "amount": { "currency": "EUR", "value": 49.99 },
          "country": "DE",
          "customer": {
            "email": "customer@example.com",
            "name": "Max Mustermann"
          }
        }
        ```

        iDEAL redirects to the customer's bank for instant confirmation. SEPA Direct Debit is pull-based — ideal for subscriptions and recurring billing.
      </Tab>
      <Tab title="Asia — Alipay & WeChat Pay">
        **Alipay** and **WeChat Pay** together cover over 90% of mobile payments in China, with growing cross-border acceptance across Southeast Asia.

        ```json Alipay
        {
          "checkout_session": "your-checkout-session-id",
          "payment_method": { "type": "ALIPAY" },
          "amount": { "currency": "CNY", "value": 500.00 },
          "country": "CN",
          "customer": {
            "email": "customer@example.com"
          }
        }
        ```

        ```json WeChat Pay
        {
          "checkout_session": "your-checkout-session-id",
          "payment_method": { "type": "WECHAT_PAY" },
          "amount": { "currency": "CNY", "value": 500.00 },
          "country": "CN",
          "customer": {
            "email": "customer@example.com"
          }
        }
        ```

        Both generate a QR code for in-app scanning. For cross-border scenarios, amounts can be quoted in the merchant's local currency and converted automatically.
      </Tab>
      <Tab title="US — ACH Bank Debit">
        ACH (Automated Clearing House) is the standard bank-to-bank payment rail in the US — lower fees than cards and ideal for high-value or recurring payments.

        ```json
        {
          "checkout_session": "your-checkout-session-id",
          "payment_method": { "type": "ACH" },
          "amount": { "currency": "USD", "value": 250.00 },
          "country": "US",
          "customer": {
            "email": "customer@example.com",
            "name": "John Smith"
          }
        }
        ```

        ACH payments settle in 1-3 business days. Use webhooks to track status transitions from `PENDING` to `SUCCEEDED`.
      </Tab>
    </Tabs>

    <Tip>
    Same API, any payment method. The only things that change between regions are `payment_method.type`, `country`, `currency`, and country-specific customer fields. See the [Country Reference](/reference/country-reference) for required fields per market.
    </Tip>
  </Step>

  <Step title="Handle the response">
    Check the payment status in the response:

    - `SUCCEEDED` — Payment completed successfully
    - `PENDING` — Awaiting confirmation (common for PIX, OXXO, PSE, and other async methods)
    - `DECLINED` — Payment was declined by the provider
    - `REJECTED` — Payment was rejected due to validation errors

    For asynchronous payment methods (PIX, OXXO, PSE, bank transfers), set up [webhooks](/guides/webhooks/setup) to receive real-time status updates.
  </Step>
</Steps>
